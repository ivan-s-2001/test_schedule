# Workforce Portal

Единый self-hosted портал для сотрудников, рабочих графиков, отпусков и внутренней документации.

## Состав

- **Frappe Framework 15** — пользователи, роли, API, уведомления и аудит.
- **ERPNext 15** — обязательная зависимость Frappe HR.
- **Frappe HR 15** — сотрудники, подразделения, типы смен, отпуска и табель.
- **Frappe Wiki 2** — внутренняя документация и история изменений.
- **Workforce Portal** — месячный редактор графиков и запросы сотрудников на изменения.
- **MariaDB 10.11** и **Redis 6.2**.

## Что реализовано в первом этапе

- месячный график по подразделению;
- строки сотрудников и колонки дней месяца;
- назначение типа смены и ручного времени;
- отображение одобренных отпусков поверх графика;
- состояния графика: черновик, опубликован, заблокирован;
- запросы на замену, перенос, задержку, сверхурочную работу и отсутствие;
- разграничение доступа между HR и сотрудниками;
- API для дальнейшего мобильного интерфейса и интеграций;
- Docker-сборка полного стека.

## Быстрый запуск в Windows

Требуются Git и Docker Desktop с включённым BuildKit.

```powershell
Copy-Item deploy/.env.example deploy/.env
./deploy/build.ps1
./deploy/start.ps1
```

После создания сайта откройте `http://localhost:8080`.

Данные администратора задаются в `deploy/.env`.

## Разработка

Приложение устанавливается в Frappe под именем `workforce_portal`.

Основной экран:

```text
/app/workforce-schedule
```

Основные DocType:

```text
Workforce Schedule
Workforce Schedule Entry
Workforce Change Request
```

## Ветки

- `main` — сохранённое исходное состояние репозитория;
- `agent/frappe-workforce-portal` — новая реализация портала.
